Job Description

You, as a Co-op Full Stack Developer, will:

  • Help to design, build, test and document modern front-end UI/UX applications in a JavaScript framework React/Redux, and functional microservices with API-driven architecture using frameworks NodeJS and Spring Boot.
  • Deploy and monitor microservices using cloud infrastructure.
  • Collaborate with architects, UX Designers, product owners, QA, and DevOps to prioritize delivering value to our customers in an Agile environment.
  • Contribute to improving product quality and the development process.

This is the right role for you if you have:

  • Strong foundations on object-oriented programming (OOPs) concepts.
  • Experience with popular frontend JavaScript frameworks like React/Redux.
  • Experience with backend frameworks such as NodeJS or Java Spring Boot.
  • Understands RESTful API implementation.
  • Working with relational databases like MySQL or NoSQL like MongoDB.
  • Excellent interpersonal and communication skills (verbal and written).
  • Experience in working with Docker and Kubernetes clusters hosted on a public cloud infrastructure (e.g., GCP, Azure or AWS).

Goals

I have attended all the stand-ups, backlog refinement meetings, and sprint planning sessions so far. As a result, I’ve learned a lot about how Agile teams operate specifically in task breakdown, estimation, and iterative development. These experiences have deepened my understanding of Agile workflows.

Goal 2: Learn the fundamentals of React by building and debugging at least 3 React components and implement Redux for global state management in at least 5 or more features.

I’ve implemented several frontend features using React and have built and debugged multiple components. This process has significantly improved my understanding of React fundamentals. I’ve also gained hands-on experience with Redux for managing global state across features.

Goal 3: Learn how to develop and optimize at least 3 API endpoints using Node.js and gain an understanding of Datadog by setting up monitors for at least 5 key metrics to track API reliability.

I’ve developed an API endpoint that implements a soft delete feature of files from GCP buckets. While I’ve started working on the backend, I still need to explore Datadog further. Setting up monitors for key metrics is an area I plan to focus on next.

Conclusion

In summary, I’ve gained a solid understanding of Agile principles by actively participating in team ceremonies. I’ve built numerous features using React and strengthened my backend development skills through hands-on experience. These skills will be invaluable in my future projects and career growth.
